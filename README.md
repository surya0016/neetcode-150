## 📂 Repository Structure

```
/problems
    /arrays

    /trees

```

* **problems/** → Contains solutions categorized by topic
* **notes/** → Contains concepts, patterns, and learnings

---

## 🧠 Approach

For each problem:

1. Understand the problem deeply
2. Try brute force first
3. Optimize step-by-step
4. Write clean and readable code
5. Add notes (intuition + patterns)

---

## ✍️ Notes Format

Each note may include:

* Problem intuition
* Approach explanation
* Time & Space complexity
* Key takeaways / patterns
